# Gardening Agent — Feature Design

> **Feature Branch:** `feat/gardening-agent`
> **Base Branch:** `feat/facade-intensity`
> **Status:** Design + MVP

---

## 1. Concept & Vision

A friendly, knowledgeable gardening assistant that lives inside the Sunlight Visualizer sidebar. The agent knows your building's facade sunlight exposures, today's weather, and — over time — your personal preferences and constraints. Rather than telling you what to grow, it *suggests with warmth*, adapts to your lifestyle (indoor/outdoor, sun/shade tolerance, watering habits), and never pushes. Think of it as a wise neighbor who happens to have a horticulture degree and genuinely cares about your success.

## 2. Feature Inventory

### 2.1 Chat Window (`GardeningChat.tsx`)

- Collapsible panel docked at the bottom of the sidebar (above/below the existing panels)
- Toggle button: "🌱 Gardener" with an unread message dot
- Header: agent name "Fern" + status line ("here to help", "checking your facade…")
- Message history: scrollable, user messages right-aligned (teal), agent messages left-aligned (sage green), with avatar initials
- Input row: auto-growing textarea + send button
- Typing indicator: animated dots while awaiting response
- Persisted to `localStorage` as `gardener_chat_history` (last 50 messages)

### 2.2 Weather Integration (`weatherService.ts`)

- Fetches from **Open-Meteo API** (free, no key required): `https://api.open-meteo.com/v1/forecast`
- Called when location changes OR when user manually refreshes
- Parameters: `latitude`, `longitude`, `daily=temperature_2m_max,temperature_2m_min,precipitation_sum,weathercode`, `current_weather=true`, `timezone=auto`
- Cache result in `localStorage` as `gardener_weather_cache` with 30-minute TTL
- Data shape:
  ```typescript
  interface WeatherData {
    location: LatLng;
    fetchedAt: Date;
    current: { temp: number; code: number; windspeed: number; isDay: boolean };
    today: { high: number; low: number; precipitationMm: number; condition: string };
    forecast: Array<{ date: string; high: number; low: number; precipitationMm: number; condition: string }>;
  }
  ```

### 2.3 Plant Database (`plantDatabase.ts`)

- Client-side JSON plant database: ~80 common edible/ornamental plants
- Each entry:
  ```typescript
  interface PlantEntry {
    id: string;
    name: string;
    emoji: string;
    category: 'vegetable' | 'herb' | 'fruit' | 'flower' | 'succulent';
    sunRequirement: 'full' | 'partial' | 'shade';
    minDirectSunHours: number;    // hours of direct sun needed on facade
    heatTolerance: 'cool' | 'moderate' | 'heat-loving';
    frostTolerance: 'none' | 'light' | 'hardy';
    waterNeeds: 'low' | 'moderate' | 'high';
    daysToHarvest?: number;
    beginnerFriendly: boolean;
    notes: string;
  }
  ```
- Matching logic: `matchPlants(facadeExposure, weather, userPreferences) → PlantEntry[]`

### 2.4 User Preferences (`userPreferences.ts`)

- Stored in `localStorage` as `gardener_preferences`
- Shape:
  ```typescript
  interface GardenerPreferences {
    displayName?: string;         // how the agent addresses the user
    gardenLocation: 'indoor' | 'outdoor' | 'both';
    experience: 'beginner' | 'intermediate' | 'expert';
    waterFrequency: 'daily' | 'few-days' | 'weekly' | 'forgetful';
    petsOrKids: boolean;          // avoid toxic plants if true
    goals: string[];             // e.g. ['fresh herbs', 'saves money', 'pretty flowers']
    avoidPlants: string[];       // plant IDs user has had bad experiences with
    favoriteSeasons: ('spring' | 'summer' | 'fall' | 'winter')[];
    lastUpdated: string;          // ISO date
  }
  ```
- Preferences are **never assumed** — the agent asks gently and updates

### 2.5 The Gardener Agent (`GardenerAgent.ts`)

- Stateless class that takes current context + chat history and returns an agent prompt string
- Prompt engineering: warm, encouraging, curious, non-prescriptive tone
- Tool use: `weatherService`, `plantDatabase`, `facadeExposures`, `userPreferences`
- Conversation flow:
  1. On open: greet naturally, mention facade highlights or today's weather
  2. On plant query: match plants → explain why they fit their specific facade + weather
  3. On preference learning: ask one question at a time, never interrogate
  4. On unexpected input: graceful redirect, never dismiss
- Response is generated by the **Hermes LLM** via the parent app's `delegate_task` mechanism
- The agent generates structured JSON that the ChatWindow component renders

### 2.6 Plant Suggestions Panel (`PlantSuggestions.tsx`)

- Appears when the agent identifies 3+ suitable plants for a facade direction
- Shows cards: plant emoji + name, sun requirement badge, difficulty badge, harvest time
- Click card → expands to full plant info + agent's reasoning
- "Save to favorites" button → adds to a mini garden list
- Hidden when no plants match well enough

### 2.7 Gardener Memory (`gardenerMemory.ts`)

- Stored in `localStorage` as `gardener_memory`
- Accumulates: plants user has tried, outcomes (success/failure notes), preferences stated over time
- Shape:
  ```typescript
  interface GardenerMemory {
    plantHistory: Array<{ plantId: string; outcome: 'success' | 'failure' | 'unknown'; notes: string; seasonPlanted: string }>;
    conversationThemes: string[];   // topics already discussed
    lastInteraction: string;        // ISO date
    interactionCount: number;
  }
  ```
- The agent prompt includes a memory summary so context carries across sessions

## 5. Gardener Agent Prompt Strategy

The agent uses a **structured output** approach:

1. User message → `GardenerAgent.generatePrompt(context)` → prompt string
2. Prompt sent to Hermes LLM via `delegate_task`
3. LLM returns JSON:
   ```json
   {
     "message": "Hello! I see your south-facing wall gets about 7 hours...",
     "suggestedPlants": ["tomato-01", "pepper-01"],
     "prefQuestion": null,
     "memoryUpdate": null
   }
   ```
4. `GardenerChat` renders `message`, shows plant cards for `suggestedPlants`, and if `prefQuestion` is set, shows a quick-interaction inline form

**Agent tone rules:**
- Never say "you should" — say "you might enjoy" or "based on what you've told me"
- When uncertain, ask a question instead of guessing
- Celebrate small wins ("Woo! Cherry tomatoes! 🎉")
- Reference specific data: "Your SW wall gets the most afternoon sun — perfect for heat-lovers"
- Ask before assuming: "Before I suggest anything, do you prefer herbs for cooking or more flowers?"

## 6. Weather → Plant Matching Rules

| Weather Factor | Effect |
|---|---|
| Current temp > 30°C + low precipitation | Prioritize heat-loving, drought-tolerant plants |
| Current temp < 10°C | Focus on cool-season crops, warn about frost-sensitive plants |
| Rain forecast > 5mm today | Mention: "Good day to transplant — rain will settle the soil!" |
| Cloudy week ahead | Suggest shade-tolerant plants more prominently |
| UV index > 8 | Warn about sensitive plants, suggest afternoon shade |

---

## 7. Facade → Plant Matching Rules

| Facade | Best For |
|---|---|
| S / SW / SE | Full-sun vegetables, heat-loving herbs, fruiting plants |
| E | Morning-sun crops, cool-season in hot climates |
| W | Afternoon-sun plants, heat-tolerant varieties |
| N | Shade-loving plants, leafy greens, herbs that prefer diffuse light |
| Rooftop | Same as S but more intense — full sun, wind-hardy |

## 8. Technical Notes

- **No new backend required** — all inference is via Hermes LLM `delegate_task`
- Weather API: Open-Meteo (free, no API key, rate-limit safe with caching)
- All localStorage keys namespaced: `gardener_*`
- Chat history: capped at 50 messages, oldest pruned first
- Component lazy-loading: `GardeningChat` and `PlantSuggestions` load on demand
- The agent runs in a subagent (`toolsets: ["web", "terminal"]`) so it can fetch weather data
- GardenerAgent in the main app serializes context and delegates to the subagent for response generation
